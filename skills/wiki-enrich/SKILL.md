# wiki-enrich

周期性从 `每日/` 的日记里按主语抽取高信号内容，建议 fileback 到 `知识库/实体/`、`知识库/概念/`、`知识库/专题/` 等沉淀层目录，并汇总到 `_月度复盘/YYYY-MM.md`。这是 corpus **从流水账到沉淀**的关键步骤。

## When to trigger

- 用户说"月度复盘"、"这个月整理一下"、"从日记里抽一下"
- 用户说"enrich 一下"、"提炼最近的日记"、"periodic review"
- 月初/月末的定期维护
- 用户感觉"日记堆太多了，有用的东西淹没了"

**不要触发**：
- 单条对话要存 → `wiki-fileback`
- 单篇外部资料要存 → `wiki-ingest`
- 只是要检查健康 → `wiki-lint`

## Decision tree

1. **确定扫描范围**
   - 默认：最近一个自然月
   - 用户指定：本周 / 过去 N 天 / 某个日期区间
2. **扫日记**：读 `每日/YYYY-MM-*.md`
3. **按主语分组**（用 filing-rules）：
   - 提到的人物 → 归入对应人物
   - 提到的项目 → 归入对应项目
   - 提到的概念/方法 → 归入对应概念
4. **信号过滤**（只留高信号，避免复制整份日记）
   - 保留：决定、结论、新事实、教训、新资源
   - 丢弃：情绪流水、重复聊天、已在沉淀层的老信息
5. **对每条高信号内容，生成 fileback 提案**（**不要直接写**）
   - 目标页：`[[知识库/实体/lorekit]]`
   - 追加到 timeline 的一句话
   - 来源：`[[每日/2026-04-10]]`
6. **生成月度复盘** `_月度复盘/YYYY-MM.md`
   - 顶部：本月主要事件 / 决定 / 新资源
   - 分组：按主语列出高信号条目
   - 末尾：pending fileback 列表（供用户 Y/N 批准）
7. **等用户确认**后才执行 fileback（交给 `wiki-fileback`）
8. 运行 `wiki lint --quick` 自检

## Tools to use

- `wiki enrich --since <date>` — 主命令（v0.5+）
- `wiki search` — 辅助确认目标页是否存在
- `wiki-fileback` — 用户批准后用它实际写入（本 skill 不直接写沉淀层）
- 底层：Glob / Read / Grep

## Output format

```
=== wiki enrich 报告 ===
范围：2026-04-01 ~ 2026-04-30
扫描日记：24 篇
高信号条目：18 条

分组提案：
[[知识库/实体/lorekit]]（6 条）
  - 2026-04-02：决定方案 A 冻结老 vault，建新 corpus
  - 2026-04-10：M2 fat skill 目标 200 行上限
  - ...

[[知识库/实体/金柔柔]]（3 条）
  - 2026-04-15：阿里淘天 RAG 知识治理项目正式立项
  - ...

[[知识库/概念/fat skill]]（2 条）
  - ...

月度复盘草稿已生成：_月度复盘/2026-04.md
待批准 fileback：18 条（回复 Y 全部执行，或逐条选）
```

**铁律**：
1. 本 skill **只提案不直接写沉淀层**，实际写入交给 `wiki-fileback`
2. 信号过滤要狠，宁缺毋滥——月度复盘不是日记副本
3. 每条高信号必须带日记来源 `[[每日/YYYY-MM-DD]]`，便于追溯
